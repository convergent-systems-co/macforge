**Status:** Tech Lead plan. Ready to dispatch.
**Date:** 2026-05-21

**Story:** Apple package zip slice (issues #21–#25)

## Objective

Replace the `apple package` stub with a real implementation that produces a notarization-ready `.zip` from a Developer-ID-signed `.app`. Single `ditto` invocation. Pre-flight refuses unsigned / missing / non-`.app` inputs with structured `MF-PACKAGE-*` errors. Mirror the existing `apple/codesign/` and `apple/spctl/` patterns.

## Seed Commit (T1) — pre-flight investigation

**Conflict found, must be addressed inside T1.** The existing `internal/errors/codes.go` defines:

```go
CodePackageUnsupportedFormat = "MF-PACKAGE-001"
```

at `internal/errors/codes.go:31`. Issue #21 also pins `MF-PACKAGE-001` to `CodePackageInputNotFound`. Two codes cannot share a number.

Per codes.go's own doc-comment ("Once published, the meaning of a code never changes. Retired codes stay reserved. New conditions append; never re-use a number"), this is a `RESERVED` situation. However:

- The constant is referenced **only** in `codes.go` and `AllCodes()` (verified via `grep -rn "CodePackageUnsupportedFormat" internal/`).
- It has never been returned by any production code path.
- MacForge is pre-v0.1; the public-contract guarantee only kicks in at v1.0.

**Decision (Tech Lead):** T1 removes the unused `CodePackageUnsupportedFormat` constant. The MF-PACKAGE-001 slot is reassigned to `CodePackageInputNotFound` per the spec. T1's PR body will note the removal explicitly. If the principal wants the old constant preserved as `// RESERVED — was CodePackageUnsupportedFormat`, that lands as a follow-up PR. (Surfacing per `Common.md P3`.)

### T3 (#23) — `internal/packaging/` skeleton

**Architectural clarification.** The issue body says `func Package(rt *cliRuntime, opts Options) (Result, error)` but `cliRuntime` lives in `package main` (cmd/macforge) and cannot be imported by `internal/packaging` (import-cycle / layering violation). Following the existing `signing` / `verify` Service pattern:

- `internal/packaging/types.go`:
  - `type Options struct { Input, Output, Format string; Force bool }`
  - `type Result struct { Format, Input, Output, InputSHA256, OutputSHA256 string; SizeBytes int64; DurationMS int64 }`
  - `type Service struct { ... }` — the package-layer orchestrator
  - `func New(cs *codesign.Client) *Service` (T3 only wires codesign; T4 will add ditto via constructor option or second arg in its own PR)
- `internal/packaging/package.go`:
  - `func (s *Service) Package(ctx context.Context, opts Options) (Result, error)` — runs pre-flight, dispatches by `opts.Format`.
  - **Pre-flight checks (in order)** per spec §5:
    1. `opts.Input` exists → else `NewPackage(CodePackageInputNotFound, "packaging.Package", ...)` with `WithHint("no such file or directory: <input>")`.
    2. `opts.Input` is a directory ending in `.app` → else `NewPackage(CodePackageInputNotAppBundle, ...)`.
    3. `filepath.Join(opts.Input, "Contents", "Info.plist")` exists → else `NewPackage(CodePackageInputNotAppBundle, ...)`.
    4. `s.cs.Verify(ctx, opts.Input)` succeeds → else re-wrap as `NewPackage(CodePackageInputNotSigned, ..., WithCause(err))`.
    5. `opts.Output` doesn't exist OR `opts.Force` is true → else `NewPackage(CodePackageOutputExists, ...)`.
  - **Dispatch:** in T3, only `"zip"` recognized; for `"zip"` it returns a `Result{Format: "zip", Input, Output}` with no SHA fields populated (T4 fills them in). For any other format, return a `NewPackage(CodePackageDittoFailed, "packaging.Package", "format X not yet implemented")` *placeholder* — T4 may refine; T3 must not invent a new error code.

  - **Default output path resolution:** if `opts.Output == ""`, derive `<input-stem>.zip` next to input (`MyApp.app` → `MyApp.zip` in the same dir). Use `filepath.Join` for the directory portion. Lives in `package.go` as `func resolveOutputPath(input, output string) string`.

- `internal/packaging/package_test.go` (table-driven) covers every validation branch using `apple.FakeRunner` for the codesign verify shellout. Each case asserts the returned error's `.Code` equals the expected `MF-PACKAGE-*` constant.
- Tests use `t.TempDir()` and `filepath.Join` — no `/Library/...` or other Unix-only path literals.
- All exported identifiers carry godoc. Package doc: `// Package packaging produces .zip / .dmg / .pkg / .app artifacts from signed inputs.`

**Test note for T3 (DO NOT skip):** the unsigned-input case uses a FakeRunner that replays a `codesign --verify` exit-1 result; the test asserts the wrapped error has `Code == CodePackageInputNotSigned`, NOT the original `CodeVerifyCodesignFail`.

### T4 (#24) — `internal/packaging/zip.go` PackageZip

- `internal/packaging/zip.go` adds:
  - `Service.PackageZip(ctx context.Context, opts Options) (Result, error)` — invoked from `Package()` when `opts.Format == "zip"`.
  - Computes input SHA256 = the `.app`'s CodeDirectory hash. Implementation: shell out via `codesign.Client.Display(ctx, opts.Input)` (already exists, returns `DisplayInfo`). **Open question for tech-lead resolution before T4 dispatches:** the existing `DisplayInfo` doesn't expose CD hash. T4 either (a) extends `DisplayInfo` (touches T2's neighbor `codesign/`, NOT OWNS), or (b) parses the codesign -dvvv stderr inside packaging via a new helper. **Decision: option (b)** — parse a `CDHash=<hex>` line out of codesign's stderr inside `packaging/zip.go`. Stays inside T4's OWNS.
  - Calls `s.dt.Archive(ctx, opts.Input, opts.Output)` (where `s.dt` is the ditto Client added to Service in this PR via a constructor change — Service.New gains a `*ditto.Client` parameter. This touches `internal/packaging/types.go` which is T3 OWNS. **Resolution:** T4's PR is allowed to modify the `Service` constructor in `types.go` to accept the ditto dependency, because the *intent* of `types.go` was "the shared Service shape" and adding a second client to `New()` is a forward-only extension. Document the touch in the PR body.
  - Computes output SHA256 = `sha256.Sum256` of the produced zip bytes (re-read the file).
  - Populates `Result.{Format: "zip", InputSHA256, OutputSHA256, SizeBytes, DurationMS}`. `DurationMS` measured around the `dt.Archive` call.
  - On `dt.Archive` error: propagate (already `MF-PACKAGE-005` from T2). Do NOT re-wrap.
- `internal/packaging/zip_test.go` covers:
  - Success path with `apple.FakeRunner` returning a successful codesign-display fixture + ditto fixture; the test writes a fake-zip byte sequence to `opts.Output` (the test's pre-staging step) to make the output-SHA computation deterministic.
  - ditto failure → `MF-PACKAGE-005` propagated unchanged.
  - SHA256 computation against fixed test bytes — separate sub-test, no FakeRunner.
- `internal/packaging/zip_integration_darwin_test.go` build-tagged `//go:build darwin`:
  - Builds a tiny `Test.app` skeleton in `t.TempDir()` (Contents/Info.plist + Contents/MacOS/Test stub binary).
  - Ad-hoc-signs it via real `codesign -s -` (requires darwin; skipped when GOOS != darwin via build tag).
  - Runs `PackageZip` against it.
  - Validates the produced zip with `archive/zip` (NOT `unzip`): assert `Test.app/Contents/Info.plist` is an entry. (`unzip` may not be on every macOS CI runner; Go stdlib zip reader is.)
- Cross-platform unit tests pass on Linux/Windows; the darwin integration test is build-tag-gated.
- godoc on every exported identifier.

### T5 (#25) — `cmd/macforge/package_cmd.go` real wiring

- Replace stub `cmd/macforge/package_cmd.go` with:
  - Flags: `--output <path>` (string, empty = derive next to input), `--force` (bool).
  - `cobra.ExactArgs(1)` — exactly one input path.
  - `RunE`: builds `rt, _ := newRuntime("apple.package", true)`, `r := newRunnerWithAudit(rt)`, `svc := packaging.New(codesign.New(r), ditto.New(r))`.
  - Calls `svc.Package(cmd.Context(), packaging.Options{Input: args[0], Output: flagOutput, Force: flagForce, Format: "zip"})`.
  - Builds `packageResult{...}` which implements `output.Outputter` with `SchemaName() == "macforge.v1.apple.package"`.
  - Calls `rt.emit(result, err)`.
- Audit log emits the 4-event sequence per spec §9:
  - `signal` (start) — `op=package.start`, `note=zip`, with redacted input.
  - The `codesign --verify` and `ditto` shellouts each produce their own `invocation-attempt` + `invocation-result` pair through the existing `ExecRunner.Run` audit wiring (T5 does NOT add new audit calls for these — they ride the runner).
  - `signal` (complete) — `op=package.complete`, `artifact_sha256=<output_sha256>`.
  - The start/complete signals are new: T5 emits them via `rt.audit.Write(...)` calls around the `svc.Package` invocation. Use `audit.KindSignal`.
- `cmd/macforge/package_cmd_test.go` covers:
  - Human output rendering: `"✓ Packaged MyApp.app → MyApp.zip (3.2 MB, sha256 a1b2c3...)"`.
  - JSON envelope: assert `"schema": "macforge.v1.apple.package"` and all spec §8 fields present.
  - Error paths: one sub-test per MF-PACKAGE-* code (001–005); each surfaces with non-zero exit and the correct code in the error envelope.
  - `--force` overwrites: pre-create the output, run with `--force`, assert success and overwrite.
- `README.md` updated under "Commands" with `apple package` example, expected output, and the error-code table.
- All existing tests still pass (`make test` exit 0).
- godoc on every exported identifier.

## Out of scope (v0.2)

Verbatim from spec §10:
- `.dmg`, `.pkg`, `.app` passthrough
- Auto-sign before packaging
- Version interpolation in filename
- Code-signature quality checks beyond "is signed"
- Cross-platform builds (package is macOS-only by definition)
